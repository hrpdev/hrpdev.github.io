Hugoの設定とgithub.pagesの公開
CATEGORIES
Tags

1.hugoの設定

hugoとは

go言語製サイトジェネレーター。

homebrewのインストール

まずはhomebrewのインストールが必要。既に入ってるいる人は飛ばしてください。homebrewとはMacにソフトウェアやライブラリをインストールするパッケージ管理システム。homebrewサイト内にある以下のテキストをterminalにペースト。

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

hugoのインストール

$ brew update
$ brew install hugo

ついでにbrewをアップデート

新規サイト構築

下記のコマンドを入力すると雛形が生成されます。

$ hugo new site my-site(お好きなサイト名)

my-site
|-config.toml
|-archetypes
|-content
|-data
|-layouts
|-public
|-static
|-themes

ディレクトリ内を軽く説明

・archetypesは各ジャンルの雛形を入れるフォルダです。サイト作成時にdefault.mdファイルが生成されて書く記事に日付などをつけてくれるようになっています。

・contentは各ジャンルの記事を入れておくフォルダです。

・dataは様々なデータ入れて置くところです。（そのままです笑）

・layoutsはheaderやfotterなどの共通htmlファイルを入れて置く場所です。

・publicはビルドされファイルの出力先です。

・staticはcssや画像ファイルを入れて置く場所です。

・themesはHugoのテーマを入れて置く場所です。

この状態ではプレビューしても何も出ませんのでひとまずテーマを設定します。

テーマの設定

ひとまずthemesフォルダに移動 以下のコマンドでおそらく全テーマがローカルにクローンされる

$ git clone https://github.com/dim0627/hugo_theme_robust.git

ひとつのテーマだけをインストールしたい人はhugo themesにお洒落なテーマが沢山あるのでテーマクリック→homepageクリック→gitのclone or downloadボタンにあるコードをコピペ

$ git clone (コピペしたコードを貼り付け)

インストールしたテーマを反映させるためにはconfig.tomlに下記を追加。テーマ名はthemesフォルダ直下にできたフォルダ名。

theme="テーマ名"

これでserveすると選んだテーマが反映されます。

記事の作成

hugoの記事はマークダウン記法で書いていきます。

$hugo new post/test.md

test.mdという記事がcontent/postのフォルダ内に生成されます。

ローカルプレビュー

watch機能が付いているので変更内容がすぐに反映されます。

$ hugo serve -watch

ビルド

下記コマンドを実行するだけ。

$ cd my-site　//サイトディレクトリに移動
$ hugo //ビルドされ結果をpublicフォルダに出力

2.githubで公開

流れ

1.githubのアカウントを作る

2.リモートリポジトリの作成

3.ローカルリポジトリの作成

4.ローカルリポジトリで作成したファイルなどをリモートリポジトリにpushして公開

githubとは

githubとはソースコードのバージョン管理サービス。

githubのアカウント登録

githubのトップページでユーザー名とメールアドレス、パスワードを入力してアカウント登録をしてください。有料プランもありますがひとまず無料プランを選択。登録したアドレスに送られてくるメールから認証を行いユーザー登録完了です。

リポジトリの作成

New repositoryボタンをクリックして、リポジトリ名を入力して作成。無料プランでは全て公開になります。 この時リポジトリ名は

“アカウント名.github.io”という風にして置かないと公開されません。

ローカルリポジトリの作成とリモートリポジトリとの紐付け

hugoの場合はサイト名/publicフォルダにビルドした内容が出力されるのでpublicフォルダ内で作成してあげる必要があります。 下記のコマンドを実行します。

$ git init //ローカルリポジトリの作成

紐付けは下記のコマンドを実行。

$ git remote add origin https://github.com/アカウント名/リポジトリ名

hugoで作成したサイトをローカルリポジトリに追加

$ git add --all

-allで全て追加ファイル単位の場合は、-allをファイル名にしてあげるとおk。

ローカルリポジトリにcommit

コミットはローカルリポジトリに追加・更新などの変更記録を残すこと。

$ git commit -m"first commit"

最後にリモートリポジトリへプッシュして記事公開

$ git push origin master

上記のコマンドを実行して、ブラウザの検索窓にアカウント名.github.ioを入力すると公開されているはずです。

ここまでやってしまうと、次回からは

1.記事作成やレイアウト変更

2.ビルドする

3.ローカルリポジトリに変更をaddしてcommitする

4.リモートリポジトリにpushして公開

という簡単手順で更新作業が行えるようになります。
